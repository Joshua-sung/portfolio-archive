## Situation / Task

Company work was often shared through Notion, and visibility mattered both within the team and across other teams. However, requests were scattered across chat, documents, files, and sheets. When work started in many places, it became difficult to see ownership, current status, and required follow-up.

The task was to create a lightweight operating system for task intake, tracking, and shared visibility without adding unnecessary backend complexity. This case shows PM value in tool orchestration: using existing systems to make work easier to see and act on.

## Action

I designed the operating model around clear tool roles. Notion became the task ledger, Slack became the fast input channel, Drive became the file repository, and Sheets became the data ledger. I documented the system across 8 operating docs covering the project overview, Notion structure, Drive policy, Slack bot design, automation structure, operating rules, roadmap, and Slack-Notion Task Bot settings.

Instead of creating a separate task system, I reused the existing company Notion Tasks database. I reflected 12 confirmed Notion properties and 3 default creation values for status, priority, and work type.

For execution, I implemented a Slack mention to task title extraction to Notion task creation to original Slack thread response flow in 346 lines of Apps Script. I added a 600-second duplicate event block, empty-input guidance, thread reply behavior, and duplicate prevention so the workflow could handle common operating exceptions.

## Result

The system was deployed and used by roughly 5 direct users. Because company work was commonly shared through Notion, the broader collaboration visibility scope extended to about 27 people.

Within 3 weeks, 65 tasks were created through the system. At roughly 3 minutes saved per task, the direct input-time saving was useful, but the larger effect was better work visibility. The dashboard made ongoing work easier to share, review, and coordinate across teams.

## KPI Rationale

This project has no direct revenue metric and no approved internal hourly cost, so I lead with the defensible time-saving KPI: 65 tasks multiplied by roughly 3 minutes saved per task, or 195 minutes, equal to 3.25 hours in the first 3 weeks. The broader collaboration KPI is visibility across roughly 27 people, which explains why the system mattered beyond input-time savings.

## Tools & Tech

- Existing Notion Tasks database
- Slack mention intake flow
- Google Apps Script automation
- Google Drive file storage policy
- Google Sheets as data ledger
- Notion dashboard for shared task visibility

## Collaboration

The work required aligning operators, Notion users, Slack requesters, and cross-team viewers. I focused on fitting the automation into the team's existing work habits so the system would reduce friction instead of creating another place to maintain.

## Lessons

Workflow automation only matters if it improves visibility after the task is created. The durable value came from converting scattered requests into a shared operating surface where ownership, status, and next actions were visible.
