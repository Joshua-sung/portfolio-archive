## 상황 / 과제

7개국 7,500개 여행지에 대한 구조화 데이터 구축 프로젝트였습니다. 약 40명의 외부 작업자가 참여했고, 각 레코드에는 객관적으로 수집 가능한 정보와 사람이 판단해야 하는 주관 평가 항목이 함께 들어 있었습니다. 수작업량이 늘어날수록 건당 지급 구조 때문에 마진 압박이 커졌고, 작업자별 판단 편차도 관리해야 했습니다.

과제는 반복 입력을 줄이되 API 비용이 절감액을 상쇄하지 않도록 자동화 범위를 정하는 것이었습니다. 동시에 주관 항목의 품질이 작업자별로 흔들리지 않도록 운영 기준도 필요했습니다. 이 사례의 핵심은 자동화를 기술 과제가 아니라 마진과 품질을 동시에 지키는 데이터 운영 판단으로 설계한 점입니다.

## 실행

먼저 전체 데이터 스키마를 객관 수집 필드와 인간 판단 필드로 분해했습니다. 이 중 9개 객관 필드를 자동화 대상으로 분리했습니다. 대량 수집은 Selenium으로 처리하고, Google Maps API는 비용 대비 가치가 있는 필드에만 제한적으로 사용했습니다.

수집한 데이터는 목표 스키마에 맞게 변환한 뒤 작업자에게 배정하기 전에 선입력했습니다. 주관 평가 항목은 2점 허용 범위를 둔 평가 기준, 예시, 사전 캘리브레이션으로 편차를 줄였습니다. 검수 단계에서는 이상값을 표시하고 수정 요청을 기록했으며, 반복 오류 패턴은 다시 가이드에 반영했습니다.

주간 생산성과 오류율은 운영 대시보드로 관리했습니다. 저진척 작업자를 마감 직전이 아니라 중간에 파악할 수 있었고, 메인 PM과 클라이언트가 처리량과 품질을 같은 화면에서 볼 수 있었습니다.

## 결과

수작업 인건비를 14.8% 줄였고, 구축 예산 기준 약 327만원을 절감했습니다. 35만 건 중 19.1%를 자동 선입력해 반복 입력 업무를 직접 줄였습니다. 자동화를 전부 적용하는 방식이 아니라, 비용과 품질을 함께 고려해 마진을 방어한 운영 개선이었습니다.

## 도구와 기술

- Selenium 기반 객관 필드 수집
- Google Maps API 선택적 보강
- 스프레드시트 기반 스키마 변환과 QA
- 작업자 생산성과 오류율 대시보드
- 주관 데이터 평가 기준과 사전 캘리브레이션

## 협업

운영자, 외부 작업자, 메인 PM, 클라이언트가 연결된 프로젝트였습니다. 제 역할은 데이터 비용, 작업자 행동, 품질 리스크를 모두가 따를 수 있는 운영 시스템으로 번역하는 것이었습니다. 대시보드는 논의가 의견이 아니라 생산량과 오류 근거를 중심으로 진행되도록 만들었습니다.

## 배운 점

자동화는 워크플로우를 먼저 분해했을 때 가장 효과적입니다. 핵심은 모든 것을 자동화하는 것이 아니라, 어떤 필드는 자동화해도 안전한지, 어떤 필드는 사람의 판단이 필요한지, API 비용이 사업적으로 맞는 구간이 어디인지를 구분하는 일이었습니다.
