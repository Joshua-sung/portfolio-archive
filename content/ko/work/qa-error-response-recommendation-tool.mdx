## 상황 / 과제

검수 업무에서는 반복되는 시스템 오류를 작업자에게 이해하기 쉬운 말로 설명해야 하는 상황이 많았습니다. 기존 방식은 검수자가 오류 원문을 직접 해석하고 매번 답변을 새로 작성해야 했기 때문에, 같은 유형의 문제도 사람마다 설명 방식이 달라질 수 있었습니다.

비개발자 검수자 입장에서는 시스템 오류를 정확히 이해하기 어렵고, 작업자 입장에서는 반려 사유가 일관되지 않게 전달될 위험이 있었습니다. 과제는 반복 검수 지식을 검색 가능한 운영 자산으로 바꿔 답변 품질, 해석 부담, 처리 속도를 함께 개선하는 것이었습니다.

## 실행

이 문제를 개인 역량의 문제가 아니라 운영 지식이 구조화되어 있지 않은 문제로 봤습니다. 최신 기준으로 개발자 제공 아카이브 검수이력 43,846건을 분석했고, 이 중 에러가 포함된 이력 978건을 추출했습니다. 추출된 에러는 202개의 유니크 에러 그룹으로 정리했습니다.

다만 202개 그룹 전체를 27개로 완전히 분류했다기보다는, 반복 빈도와 실무 활용도가 높은 케이스를 중심으로 현재 27개의 대표 케이스를 에러위키에 반영했습니다. 각 대표 케이스에는 상세 답변과 짧은 답변을 구성해 총 54개의 답변 템플릿을 만들었고, 검수자가 실제 판단에 참고할 수 있도록 과거 검수자 코멘트 664건을 검색 데이터로 반영했습니다.

| 구분 | 최신 기준 |
| --- | ---: |
| 개발자 제공 아카이브 전체 이력 | 43,846건 |
| 에러로 식별된 이력 | 978건 |
| 유니크 에러 그룹 | 202개 |
| 현재 에러위키 대표 케이스 | 27개 |
| 답변 템플릿 | 54개 |
| 수정 위치 안내 데이터 | 27개 |
| 과거 검수자 코멘트 검색 데이터 | 664건 |
| 기술이슈 큐 | 5개 |
| 추가 검토 후보 | 26개 |

최근 아카이브에서 반복도가 높아 신규 대표 케이스로 반영한 항목은 이미 등록된 모상품 코드 45건, 핵심 포인트 이미지 URL 또는 확장자 오류 24건, 출발국가/도시코드 유효하지 않은 지역 15건입니다. 세 항목 합계 84건의 반복 이력을 기준으로 작업자가 자주 마주치는 오류부터 답변 자산에 우선 반영했습니다.

| 신규 대표 케이스 | 아카이브 반복 건수 |
| --- | ---: |
| 이미 등록된 모상품 코드 | 45건 |
| 핵심 포인트 이미지 URL 또는 확장자 오류 | 24건 |
| 출발국가/도시코드 유효하지 않은 지역 | 15건 |
| 합계 | 84건 |

도구는 서버나 별도 설치 없이 사용할 수 있도록 단일 HTML 형태로 만들었습니다. 검수자가 에러 원문을 붙여넣으면 유사 케이스와 작업자용 반려 문구를 확인할 수 있도록 했고, 표준 케이스가 없는 경우에는 새 패턴을 쉽게 기록할 수 있게 만들어 지식 DB가 계속 보강되도록 설계했습니다.

## 결과

성과는 개발자 제공 전체 검수이력 43,846건과 전체 운영자 수동 처리 이벤트 15,873건을 기준으로 다시 확인했습니다. 도구 도입 후 제 1인 1업무일 평균 반려 처리량은 9.81건이었고, 도입 전 전체 비교군의 5.71건 대비 +71.9%, 약 1.72배 높은 처리량을 기록했습니다.

전체 수동 검수 처리 기준으로는 31.44건에서 132.40건으로 약 4.21배 증가한 처리량이 확인됐습니다. 동시에 에러위키는 대표 케이스 27개, 답변 템플릿 54개, 수정 위치 안내 데이터 27개, 과거 검수자 코멘트 검색 데이터 664건을 갖춘 운영 자산으로 확장됐습니다. 핵심 성과는 반복 검수 지식을 개인 기억에서 꺼내, 실제 업무 중 재사용 가능한 운영 도구로 바꾼 것입니다.

## KPI 판단

이 도구는 직접 매출이나 검증된 비용 절감액이 없기 때문에 운영효율 지표 중 가장 강한 수치를 대표 KPI로 선택했습니다. 전체 수동 검수 처리량 4.21배 증가를 전면에 두고, 반려 처리량 1.72배 개선은 실제 반려 응답 업무와 더 직접적으로 연결된 보조 KPI로 배치했습니다. 43,846건의 전체 검수이력, 978건의 에러 이력, 202개 유니크 그룹은 에러위키가 일부 샘플이 아니라 실제 운영 아카이브를 기반으로 만들어졌다는 근거입니다.

## 도구와 기술

- 과거 검수 처리이력 분석
- 에러와 검수 코멘트 분류 체계
- 에러위키 대표 케이스와 답변 템플릿 라이브러리
- 과거 검수자 코멘트 검색 데이터
- 수정 위치 안내 데이터와 기술이슈 큐
- 서버 없는 단일 HTML 배포 방식

## 협업

검수자, 비개발 운영자, 개발자 제공 검수 이력이 연결된 작업이었습니다. 저는 시스템 수준의 에러 흔적을 작업자에게 전달 가능한 운영 언어로 번역하고, 반복도가 높은 오류부터 검수자가 안전하게 재사용할 수 있는 에러위키와 답변 템플릿으로 구조화했습니다.

## 배운 점

운영 지식은 검색 가능하고 재사용 가능하며 업무 중 보강하기 쉬울 때 확장됩니다. 검수 운영에서는 작은 로컬 도구라도 반복 판단의 근거를 담으면 처리 속도와 설명 품질을 함께 개선할 수 있습니다.
